Mobile Menu Test Page

This page tests the mobile hamburger menu functionality, logo positioning, and z-index stacking.

✅ LOGO FIX APPLIED: The mobile menu logo now supports both .mobile-menu-logo img and .mobile-logo class implementations for maximum compatibility.

Test Instructions:

  1. Resize your browser to mobile width (< 1024px)
  2. Click the hamburger menu button (☰)
  3. Verify the menu slides in from the right
  4. Verify the overlay covers the entire page including the header
  5. Verify the menu is fully visible (not clipped)
  6. Check logo positioning: People First logo should appear in top-left of mobile menu
  7. Check close button: X button should be in top-right, opposite the logo
  8. Test dropdown menus within the mobile menu
  9. Test at different widths: 320px, 375px, 414px, 768px
  10. Click overlay or press Escape to close

Expected Behavior:

  • Menu appears above all content (z-index: 1050)
  • Overlay appears behind menu but above content (z-index: 1049)
  • Header has z-index: 1020 (should be covered by overlay)
  • Logo positioning: People First logo in top-left corner of mobile menu
  • Close button: X button in top-right corner, properly aligned with logo
  • Responsive sizing: Logo scales appropriately (35px → 28px → 24px)
  • Body scroll is disabled when menu is open
  • Focus management works correctly
